Irgendwie ist es ein wenig an mir vorbei gegangen.
<br>
Am 17.07. wurde die Expert-Group für die neue JEE6 Spezifkation
<br>
zusammengestellt.
<br>
<br><a href="http://jcp.org/en/jsr/detail?id=316" target="_blank">http://jcp.org/en/jsr/detail?id=316</a>
<br>
<br>
Einige spannende Dinge tun sich da. Zu allererst bleibt die Einführung von "Profiles" zu bemerken. Ein recht <a href="http://douglasdooley.blogspot.com/2008/08/jee-6.html" target="_blank">kritischer Artikel</a> von Douglas Dooley beschreibt recht gut, zu welchem ersten Eindruck man kommen mag.
<br>
<br>
Ebenfalls spannend sind die deferred Teile. Nicht mehr dabei: 
<br>
<ul>
 <br>
 <li>JSR-168 Portlet Specification</li>
 <br>
 <li>JSR-170 Content Repository for Java technology API</li>
 <br>
 <li>JSR-207 Process Definition for Java</li>
 <br>
 <li>JSR-208 Java Business Integration (JBI)</li>
 <br>
 <li>JSR-225 XQuery API for Java (XQJ)</li>
 <br>
 <li>JSR-235 Service Data Objects</li>
 <br>
 <li>JSR-286 Portlet Specification 2.0</li>
 <br>
 <li>JSR-289 SIP Servlet v1.1</li>
 <br>
 <li>JSR-301 Portlet Bridge Specification for JavaServer Faces</li>
 <br>
</ul>
<br>
Als Trost bleibt: Vielleicht kommen die mit JEE7 und dem <a href="http://jcp.org/en/jsr/detail?id=277" target="_blank">JSR-277 Java Module System</a> wieder.
