Es ist so weit. Die neue Portlet Spezifikation 2.0 (<a href="http://jcp.org/en/jsr/detail?id=286" target="_blank">JSR 286</a>) ist da. Lang hat es gedauert und es scheint auch ein steiniger Weg gewesen zu sein, die Ideen von Sun und IBM unter einen Hut zu bekommen.
<br>
<br>
<ul>
 <br>
 <dl>
  The major changes in version 2 of the Java Portlet API are:
 </dl>
 <br>
 <li>Events - enabling a portlet to send and receive events and perform state changes or send further events as a result of processing an event.</li>- Public render parameters 
 <br>
 <li>allowing portlets to share parameters with other portlets.</li>
 <br>
 <li>Resource serving - provides the ability for a portlet to serve a resource (including doing ajax calls).</li>
 <br>
 Portlet filter - allowing on-the-fly transformations of information in both the request to and the response from a portlet. 
 <br>
</ul>
<br>
